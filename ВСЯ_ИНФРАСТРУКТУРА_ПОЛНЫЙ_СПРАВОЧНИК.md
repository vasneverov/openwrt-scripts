## 1. ЖЕЛЕЗНЫЕ ПРАВИЛА (Iron Rules)

### 1.1 Tailscale — спасать любой ценой
- `fw_mode=none` — иначе Tailscale убивает маршрутизацию
- `init.d tailscale DISABLED` — иначе при ребуте Tailscale стартует раньше сети
- `exclude_ntp=1` — иначе NTP ломает синхронизацию
- `rc.local` с tailscaled (sleep 40 для WR3000H/TR3000, sleep 25 для M3000)
- Watchdog в crontab (каждые 3 минуты проверяет tailscaled)
- `autoupdate=false` — никогда не обновляться автоматически
- **Запрещено:** `tailscale serve` в rc.local (блокирует SSH навсегда)
- **Запрещено:** `--state=` вместо `--statedir=` (ломает авторизацию после ребута)
- **Запрещено:** TUN-режим без `--tun=userspace-networking` (kernel panic)

### 1.2 YT профиль — только заглавными
- `podkop.YT` — заглавные буквы
- Никогда не создавать `podkop.yt` (строчные)
- Community list: только `youtube`

### 1.3 Main профиль — не трогать
- Без явного разрешения пользователя Main не менять
- Даже если ключ кажется нерабочим — сначала спросить

### 1.4 Podkop не перезагружать
- Только сохранить (`uci commit podkop`)
- Пользователь сам нажмёт Save & Apply или перезагрузит

### 1.5 Ключи проверять перед установкой
- `python3 ~/CLAUDECODE/check_vless.py <ключ>`
- Только если `● READY ✓✓✓` — ставить на роутер

### 1.6 SSH к OpenWrt
- Всегда флаги: `-o PreferredAuthentications=password -o PubkeyAuthentication=no`
- Иначе sshpass падает exit 255

### 1.7 Не перезагружать роутер без подтверждения
- Особенно если роутер за 1000 км от пользователя

### 1.8 Один UUID на оба профиля (YT + Main)
- Меньше путаницы, проще диагностика

### 1.9 REALITY ключи — только через xray x25519
- `/usr/local/x-ui/bin/xray x25519`
- ❌ openssl даёт неверные ключи
- gRPC Reality: поле flow должно быть ПУСТЫМ

### 1.10 REALITY ключи — только Apple.com и Chrome
- **SNI всегда `www.apple.com`** — его никто не блокирует
- **FP (fingerprint) всегда `chrome`** — никакого firefox
- Никогда не придумывать свои sni/fp — всегда брать из рабочего inbound

### 1.11 Новый inbound = копировать настройки с рабочего
- Не создавать inbound "с нуля" — всегда брать за основу существующий рабочий
- sniffing всегда `enabled: true` с `destOverride: ["http","tls","quic","fakedns"]`
