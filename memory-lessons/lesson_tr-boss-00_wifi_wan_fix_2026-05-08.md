# Урок: tr-boss-00 (100.123.243.33) — WiFi WAN, сломал интернет rescue_generic.sh

## Дата: 2026-05-08

## Проблема
Применил `rescue_generic.sh` на роутер tr-boss-00 (100.123.243.33).
Скрипт добавил `network.wan.ifname=eth0`, но роутер брал интернет **по воздуху** (WiFi-мост к другому роутеру), а не через WAN-порт.

**Результат:** Интернет полностью пропал, потому что podkop начал использовать eth0 как WAN, но eth0 физически никуда не подключён.

## Корень
- Роутер работает в режиме WiFi-моста (STA-режим) — интернет приходит через WiFi-интерфейс (wlan0/phy0-sta0)
- WAN-порт (eth0) не используется для интернета
- `rescue_generic.sh` (шаг 3) бездумно добавляет `network.wan.ifname=eth0`, что ломает конфигурацию

## Что нужно делать в будущем
1. **Перед применением rescue_generic.sh проверять WAN-интерфейс:**
   ```
   ip route show default
   uci show network.wan
   ```
2. Если default route идёт через WiFi (не eth0) — **НЕ добавлять** `network.wan.ifname`
3. В rescue_generic.sh нужно добавить проверку: если WAN уже работает через не-eth0 интерфейс — пропускать шаг 3
4. Для роутеров с WiFi WAN нужно отдельное решение: определить реальный WAN-интерфейс и указать его в `network.wan.ifname`
