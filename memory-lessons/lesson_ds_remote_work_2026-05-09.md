# Урок: Удалённая работа через DeepSeek на PL5 (09.05.2026)

## Что сделали
Создали полноценную систему удалённой диагностики и ремонта роутеров через DeepSeek V4 Flash API на сервере PL5 (Польша).

## Архитектура

```
Телефон (Termius) → PL5 (91.92.46.229) → Tailscale → Роутер
                        ↕
                     GitHub (vasneverov/router-lab)
                        ↕
                   Компьютер (VS Code + Cline)
```

### Компоненты:
1. **`ds.py`** — Cline-подобный интерфейс на Python, работает через DeepSeek V4 Flash API
2. **`ds.sh`** — точка входа (просто `ds` в консоли)
3. **`/root/router-lab/`** — git-репозиторий, синхронизированный с GitHub
4. **`sync-routerlab.sh`** — двухсторонняя синхронизация компьютер ↔ GitHub ↔ PL5

## Как работает `ds.py`

### При старте:
1. Синхронизация с GitHub (git fetch + reset --hard)
2. Загрузка контекста: IRON_RULES.md, deepsick_memory.md, последний урок
3. Всё передаётся DeepSeek как system prompt

### В цикле:
1. Пользователь вводит запрос
2. DeepSeek отвечает командами: `/bash`, `/read`, `/write`
3. `ds.py` выполняет команды, показывает результат
4. Результат отправляется обратно DeepSeek для анализа
5. Цикл повторяется, пока DeepSeek не выдаст итоговый вывод без команд

### При выходе:
1. Автосохранение в GitHub (git add + commit + push --force)

## Ключевые находки

### 1. DeepSeek нужно явно указывать формат команд
Без инструкции он пишет в markdown (```bash). С инструкцией — использует `/bash`, `/read`, `/write` в чистом виде.

### 2. Контекст при старте — критически важен
Без IRON_RULES и deepsick_memory DeepSeek не знает:
- Что Tailscale нельзя трогать
- Какой пароль у роутеров
- Какая архитектура инфраструктуры
- Что делали в прошлый раз

### 3. sshpass + MASTER_CREDENTIALS.md решает проблему паролей
DeepSeek сам читает файл с паролями и использует sshpass. Не нужно спрашивать пользователя.

### 4. Цикл итераций — must have
Без цикла DeepSeek выполняет одну команду и завершает. С циклом (до 10 итераций) может:
- Сделать ping → проанализировать → сделать ssh → проанализировать → выдать итог

### 5. PL5 — идеальный шлюз
- Видит все роутеры через Tailscale (30+ устройств)
- Телефону не нужно переключать учетки Tailscale
- Достаточно зайти на PL5 через Termius

### 6. GitHub как единое хранилище
- Все изменения с телефона сохраняются в GitHub
- При возвращении к компьютеру — Cline подхватывает актуальную версию
- Двухсторонняя синхронизация: что сделано на компе → доступно с телефона и наоборот

## Проблемы и решения

### Проблема: DeepSeek просит пароль
**Решение:** Добавить в system prompt инструкцию читать MASTER_CREDENTIALS.md и использовать sshpass.

### Проблема: DeepSeek не анализирует результаты команд
**Решение:** Добавить цикл итераций. После выполнения команд отправлять результат обратно DeepSeek с запросом "проанализируй результаты".

### Проблема: DeepSeek пишет команды в markdown
**Решение:** В system prompt явно указать: "Не используй markdown-форматирование. Используй ТОЛЬКО команды /bash, /read, /write в чистом виде."
