# lesson: nightly_daily_fallback — 2026-05-12

## Проблема
Если пользователь пишет заметки после полуночи (например в 1:00 ночи), `date +%Y-%m-%d` показывает уже новый день. `process.sh` проверяет `daily/$TODAY.md` — он пустой (только `# 2026-05-13`), и выходит с `exit 0` (пропуск обработки).

Заметки записаны в `daily/2026-05-12.md`, но бот их не видит.

## Решение
1. Добавлена переменная `YESTERDAY=$(date -d "yesterday" +%Y-%m-%d)`
2. Если `daily/$TODAY.md` пустой (<50 байт) — проверяется `daily/$YESTERDAY.md`
3. Если вчерашний файл есть и непустой — используется он
4. Добавлена `DAILY_REL` — относительный путь к daily-файлу (меняется при фоллбэке)
5. Все промпты DeepSeek используют `$DAILY_REL` вместо хардкода `daily/$TODAY.md`

## Изменённые файлы
- `/home/sbrain/sbrain/scripts/process.sh` — строки 28-70 (блок ORIENT), промпты

## Логика
```
TODAY=2026-05-13 → daily/2026-05-13.md пустой
  → проверяем daily/2026-05-12.md — есть записи
  → DAILY_REL="daily/2026-05-12.md"
  → DeepSeek читает daily/2026-05-12.md
```
