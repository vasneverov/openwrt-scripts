# Урок: Протокол диагностики роутеров — установлено 11.05.2026

## Правила диагностики

1. **Не создавать скрипты под каждый роутер.** Использовать универсальный скрипт из скилла `router-full-report` через SSH pipe напрямую.
2. **Не смотреть базы (ROUTERS_REPAIRED_BASE, ROUTERS_GROOMED_BASE) перед диагностикой.** Диагностика сама покажет hostname и всё нужное.
3. **Формат отчёта — строго 7 секций** (Karpathy-стиль): Провайдер, Прокси, Сайты, Tailscale, Сторожевые, Что не идеально, Итог.
4. **После отчёта — план лечения** (что, как, зачем).
