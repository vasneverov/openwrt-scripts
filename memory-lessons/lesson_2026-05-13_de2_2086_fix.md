# DE2:2086 fix — 3x-ui bug + xray-direct

## Проблема
Inbound 2086 на DE2 (195.26.231.228) не работал — TLS handshake падал с `SSL: None`.

## Причина
3x-ui (версия 3.0.1+) хранит stream_settings в БД в своём формате (не JSON). При генерации config.json 3x-ui добавляет лишнее поле `settings` внутри `realitySettings`:
```
realitySettings: {
  dest: www.wildberries.ru:443,
  serverNames: [www.wildberries.ru],
  privateKey: ...,
  shortIds: [9fee82cd],
  settings: {           ← ЛИШНЕЕ! 3x-ui баг
    publicKey: ...,
    fingerprint: chrome,
    serverName: ,
    spiderX: /
  }
}
```
xray не понимает это поле и игнорирует realitySettings целиком → streamSettings становится null.

## Решение
1. Остановить x-ui: `systemctl stop x-ui`
2. Создать правильный config.json вручную (без поля `settings`)
3. Запустить xray напрямую через systemd сервис `xray-direct.service`

## Создан systemd сервис
`/etc/systemd/system/xray-direct.service`:
```
[Unit]
Description=Xray Direct (without x-ui)
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/x-ui/bin/xray-linux-amd64 -c /usr/local/x-ui/bin/config.json
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

## Важно
- x-ui при старте перезаписывает config.json из своей БД (с багом)
- Поэтому x-ui должен быть **остановлен**, работает только xray-direct
- Если нужно править inbound — через SQLite: `/etc/x-ui/x-ui.db` (таблица `inbounds`)
- Но UPDATE в БД не помогает — 3x-ui всё равно генерирует кривой config.json

```bash
systemctl start x-ui
# После x-ui нужно снова остановить его и запустить xray-direct
```
