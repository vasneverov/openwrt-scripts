# Проверка серийного номера при прошивке M3000

## Команда для проверки

```bash
ssh -o StrictHostKeyChecking=no root@192.168.1.1 "cat /proc/device-tree/serial-number 2>/dev/null || cat /tmp/sysinfo/board_serial 2>/dev/null || dmesg | grep -i 'machine model\|serial' | head -3"
```

## Интерпретация

| Серийник | Версия | Действие |
|----------|--------|----------|
| Начинается на **"24"** | v1 (RTL8211 PHY) | ❌ Не шить! Нужна v1 прошивка |
| Начинается на **"25"** | v2 (Motorcomm YT8821) | ✅ Шить v2-yt8821 с force upgrade |
| Начинается на **"26"** | v3? | ⚠️ Уточнить перед прошивкой |

## Двойная проверка

1. **Наклейка на коробке** — должна быть "V2.0" в конце
2. **Серийник через SSH** — должен начинаться на "25"

Если не совпадает — стоп и уточнение!

## Интеграция в Шаг 0 прошивки

```bash
# Шаг 0a: Проверка модели
MODEL=$(sshpass -p '' ssh root@192.168.1.1 "cat /tmp/sysinfo/model" 2>/dev/null)
echo "Модель: $MODEL"

# Шаг 0b: Проверка серийника
SERIAL=$(sshpass -p '' ssh root@192.168.1.1 "cat /proc/device-tree/serial-number" 2>/dev/null)
echo "Серийник: $SERIAL"

# Шаг 0c: Подтверждение v2
if echo "$SERIAL" | grep -q "^25"; then
    echo "✅ Подтверждено: v2 (Motorcomm PHY) — шить с force upgrade"
    V2_CONFIRMED=1
else
    echo "⚠️ Серийник не начинается на 25 — проверить наклейку!"
    V2_CONFIRMED=0
fi
```

## Пример M56-13 (сегодня)

- Наклейка: **V2.0** ✅
- Серийник: **250512...** ✅ (начинается на 25)
- Решение: force upgrade v2-yt8821 ✅
- Результат: WAN работает ✅

## Завтра (10 банок)

Перед каждой банкой:
1. Смотрим наклейку → V2.0?
2. Подключаемся → читаем серийник → на 25?
3. Обе проверки ✅ → force upgrade v2

**Why:** Двойная проверка исключает ошибку (наклейка может быть от другой коробки)
**How to apply:** Шаг 0 прошивки всегда включает чтение серийника
